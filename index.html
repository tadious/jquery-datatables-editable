Editable DataTable example

Preamble

DataTable Editable plugin enchances standard DataTables plugin by adding functionalities that enables a user to edit cells inline, select and delete rows. All required processing is encaplulated within the plugin, and only required thing is a server-side code that accepts add, update and delete requests.

In the basic usage of DataTables Editable plugin, user can select and delete record in table and edit cells inline by double-clicking on the cell

On the server-side you will need to create one page that handles delete AJAX requests (called DeleteData.php in this example). This page should accept one parameter id. This parameter is id of the record that that should be deleted (id is placed in tag that surrounds the cell user edited). Server-side page should return string "ok" if a record is successfully deleted or error message if delete failed.

Also you will need to create one page that handles update AJAX requests (I called it UpdateData.php). This page should accept following parameters:

  1. value - that contains text used entered in the table cell while editing
  2. id - id of the record that is deleted (id is placed in tag that surrounds the cell that has been edited)
  3. columnId - position of the column of the cell that has been edited (hidden columns are counted also)
  4. columnPosition - position of the column of the cell that has been edited (hidden columns are not counted)
  5. rowId - id of the row containing cell the cell that has been edited
Server-side page should return value that is updated if cell is successfully or error message.

Initialisation code

In the initialization code you will need to pass URL of the pages that accept delete and update requests on the server side.

$(document).ready( function () {
		$('#example').dataTable().makeEditable({
							sUpdateURL: "UpdateData.php",
                    					sDeleteURL: "DeleteData.php"
							});
}

Additional HTML code

One delete button should be placed in the HTML source. This button will be enabled each time user selects a row in the table and disabled each time user deselect row. Id of the button must be "btnDeleteRow".

 <!-- Delete button can be placed anywhere in the page -->
<button id="btnDeleteRow">Delete</button>

This plugin tries to find id of the row in the id attribute of the <tr> tag. Therefore, rows in the table must be in the following format:

		<tr id="2">
			<td>Misc</td>
			<td>Links</td>
			<td>Text only</td>
			<td class="center">-</td>
			<td class="center">X</td>
		</tr>

Server-side code for delete request (PHP)

Server-side page that accepts delete request should return "ok" value is row is deleted on the server-side, or an error message if delete fails. In this example server-side page always returns "ok" value:

<?php
	//Always accepts delete
	echo "ok"
?>

Server-side code for update request (PHP)

Server-side page that accepts update request should return the same value that plugin sent in the value request parameter if record is updated on the server-side, or an error message if update fails. In this example server-side page always returns posted value:

<?php
	$editedValue = $_GET['value']
	//Always accepts update(return posted value)
	echo $editedValue
?>
